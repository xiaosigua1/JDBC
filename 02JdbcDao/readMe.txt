 遗留的问题：
   01.getConncetion还可以加if判断
   02.关闭资源没有优化
   03.所有的增删改都是一个方法executeUpdate()
   04.所有的查询都是一个方法executeQuery()
   05.如果参数多 而且属于一张表，是用javaBean

     
   
优化1：
   
     关闭资源
     因为所有的数据库链接在使用之后必须关闭！不然浪费资源！
 

优化2： 
  
   优化所有的 增删改！  
     用户有增删改！ 商品有增删改！新闻也有商品有增删改！.........
   
   增删改的sql语句 无外乎就是有和没有参数！！！
   
   delete  from  user;
   delete  from  user  where id=?;
  
   你知道用户是否给我们传递参数了吗？
   如果用户传递了参数？你知道参数的类型吗？
  
优化3：   

   用户有查询！ 商品有查询！新闻也有查询！.........
   
   select  * from user;
   select  * from user  where id=?  and name=? ;
   
优化4：

    把方法的参数提取成javaBean
    
    01.如果方法中的参数比较多并且属于一张表的信息时，
         提取成一个实体bean!
    02.实体bean对应的就是数据库中的表！
    03.把代码中用到的多个参数换成实体类
   
  如果不这么做： 产生的问题
  01. 可读性差
  02. 不利于我们后期的修改和维护
  03. 不利于代码复用
   
   
  Dao模式！ 主流的数据访问模式！
   
 Dao（Data Access Object）   数据存取对象 
     01. 实现对持久化数据的访问
		                   持久化数据在哪里？===>数据库
		                   实现对数据库的访问   
     02. 位于业务逻辑层和持久化数据之间 ===》实体类
                                实体类对应数据库中的表！
                                实体类中的属性对应数据库中表的列！
     03. 起着转换器的作用！
			      问题？
			  
			  怎么把实体类中的属性赋值给数据库中表的列？
			  
			  中间是不是需要一个转换的工具？？？
			  这个工具就是Dao!
			  
     
     
     例子：
     
   有实体类                                                       数据库
 Easybuy_User                      表1===》user
 Easybuy_News                      表2===》news
     
 
dao模式的组成部分：
  01.dao接口
  02.dao实现类
  03.实体类
  04.数据库连接需要的工具类
